Repository

A repository is a place where files are stored and tracked by git.

You can create one in any folder by typing git init

You can also clone a repository

l  A clone copies a remote repo to your local computer

l  Sometimes I create a repo locally, and clone it to another computer

 

Identifying yourself to git

git config --global user.email “you@example.com”

git config --global user.name “Your name”

 

You do this once per system, not once per repository

For this class, it must be your umich email

 

Every SSH session

l  export EDITOR=nano

 

git status

git status will tell you the current status of your current repo

 

git add

While not a git thing directly, you tell git you want to add a thing to be tracked.

You might have a large number of files, but you might not want to track ones that contain sensitive information.

git add filename

or git add directory/*

 

git commit

$ git commit [file1] [file2] ... -m [message]

A commit takes a set of changes and tracks them in the repository.

Every commit had a

l  An id

l  A message

l  Who did it

l  When it was done

l  What was changed

No more than 50 characters.

Use the imperative.

This is your documentation.

If you have a tracking id include it.

Short and to the point.

Never 1 word.

 

git remote

l  A “remote” in git defines a location that you can send copies of all or part of your repository.

l  A common remote location is github.

l  Your remote location will give you the syntax here.

l  Your repository can go to more than one remote.

 

git push

l  When we are happy with our local repository, we can push it to a remote.

l  This kind of like a copy.

l  We need to tell git which remote we want to push it to, and what branch/tag we want to push.

l  git push origin main

n  origin is the name of our remote.

n  Main is the branch

 

Branch

l   Every change you make is tied back to a branch.

l   Every repository has a primary branch.

l   Historically, it has been called master.

l   Recently, there has been a movement to rename master to main.

l   You should use main as your primary branch.

l   git branch -M main

 

git tag

l   Assign a human readable version or tag to a commit

l   Makes it easy to tell people what point to use a repo.

l   Useful for releases.

n  This will open up nano and have you edit your commit message.

l  git status

n  This will show one untracked file

l  git add demo.txt

l  git commit demo.txt

l  git status

Create a new repository on the command line

echo “#test” >> README.md

git add README.md

git commit -m “first commit”

git branch -M main

git remote add origin git@github.com:mlhess/test.git

git push -u origin main

 

or push an existing repository from the command line

git remote add origin git@github.com:mlhess/test.git

git branch -M main

git push -u origin main

 

You can reset the url if repository is not found

git remote set-url origin git@github.com:mlhess/test.git
